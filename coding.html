🔍 XPath — Find the Right Locator

Use text() for exact match or normalize-space() if there might be surrounding whitespace.

// Exact text match //button[text()='Submit'] // Trim whitespace (safer) //button[normalize-space()='Submit'] // Any element with that text //*[text()='Submit'] // In Java driver.findElement(By.xpath("//button[normalize-space()='Submit']"));
Tricky Edge
If the button contains an inner <span>, text() won't see it. Use contains(., 'Submit') which checks the full concatenated text of the element and all descendants.

XPath @class="menu active" is an exact, full-string match. If the element has class "nav menu active visible", the attribute value is not exactly "menu active" so nothing is found.

// BAD — breaks if any other class is present //div[@class="menu active"] // GOOD — check for individual tokens //div[contains(@class,'menu') and contains(@class,'active')] // SAFEST — space-delimited contains (avoids "submenu" false match) //div[contains(concat(' ', normalize-space(@class), ' '), ' menu ') and contains(concat(' ', normalize-space(@class), ' '), ' active ')] // CSS alternative (cleanest) div.menu.active

One of the most common XPath mistakes in interviews and real projects. Always use contains(@class,'token') or CSS class selectors.

Locator Strategies — Finding Elements Every Possible Way

All possible strategies to locate elements in Selenium WebDriver.

Uses the HTML id attribute. Must be unique on the page. Fastest locator strategy.

WebElement el = driver.findElement(By.id("username")); el.sendKeys("admin");

Matches the name attribute — commonly used with form inputs.

WebElement pwd = driver.findElement(By.name("password")); pwd.sendKeys("secret");

Matches elements that have the specified CSS class. Only one class at a time — no spaces allowed.

WebElement btn = driver.findElement(By.className("submit-btn")); List<WebElement> cards = driver.findElements(By.className("product-card"));

List<WebElement> links = driver.findElements(By.tagName("a")); List<WebElement> inputs = driver.findElements(By.tagName("input")); WebElement body = driver.findElement(By.tagName("body"));

// Exact match — case sensitive driver.findElement(By.linkText("Forgot Password?")).click();

// Partial match — useful for dynamic text driver.findElement(By.partialLinkText("Forgot")).click(); driver.findElement(By.partialLinkText("Sign")).click(); // matches "Sign In" or "Sign Up"

Faster than XPath in most browsers. Supports attribute, pseudo-class, sibling combinators.

// By attribute value driver.findElement(By.cssSelector("input[type='submit']")); // Child combinator driver.findElement(By.cssSelector("form.login > button")); // Multiple classes driver.findElement(By.cssSelector(".btn.btn-primary")); // nth-child driver.findElement(By.cssSelector("table tr:nth-child(2) td:first-child")); // Attribute starts-with (^=), ends-with ($=), contains (*=) driver.findElement(By.cssSelector("a[href^='/product']")); driver.findElement(By.cssSelector("img[src$='.png']")); driver.findElement(By.cssSelector("div[class*='card']"));

// Absolute (fragile — avoid) driver.findElement(By.xpath("/html/body/div[1]/form/input[2]")); // Relative (preferred) driver.findElement(By.xpath("//input[@id='email']")); driver.findElement(By.xpath("//button[text()='Submit']")); driver.findElement(By.xpath("//button[contains(text(),'Sign')]")); driver.findElement(By.xpath("//input[starts-with(@name,'user')]")); driver.findElement(By.xpath("//div[@class='card' and @data-id='5']"));

// parent axis driver.findElement(By.xpath("//input[@id='email']/parent::div")); // following-sibling driver.findElement(By.xpath("//label[text()='Email']/following-sibling::input")); // preceding-sibling driver.findElement(By.xpath("//button[text()='Save']/preceding-sibling::input[1]")); // ancestor driver.findElement(By.xpath("//span[@class='error']/ancestor::form")); // descendant driver.findElement(By.xpath("//div[@id='nav']//a[text()='Home']")); // following (all elements after current node) driver.findElement(By.xpath("//h2[text()='Section 2']/following::p[1]"));

Use this priority order for stable, maintainable tests:

  1. By.id — fastest, always prefer if unique
  2. data-testid / data-cy — purpose-built for automation
  3. By.name — great for form inputs
  4. By.cssSelector — fast, readable, broadly supported
  5. By.xpath — powerful but slower; use for complex traversal
  6. By.className / By.tagName — use only when unique
  7. By.linkText / By.partialLinkText — links only
  8. Avoid: absolute XPath, brittle CSS paths
